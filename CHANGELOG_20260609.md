Updated **TranzorExporter.exe** (v20260609)

What's new in 9 June update:

*This release re-aligns the desktop app with Tranzor's new platform login — the API now enforces **Bearer-JWT** authentication server-side, so signing in is required before any data can load. On top of that it ships a new Pre-Translation Check tab, Full Email review tooling, and a faster, more complete post-edit pipeline.*

- 🔐 **Mandatory sign-in — Tranzor 登录鉴权适配** — Tranzor's platform now enforces **Bearer-JWT authentication** on its API (a server-side change); the desktop app previously sent no credentials, so every data load returned `401 — Failed to load task data`. On startup the app now shows a modal **LDAP sign-in** and injects `Authorization: Bearer <token>` on **platform hosts only** — covering every module's session *and* bare `requests` calls, while leaving GitLab untouched. The 7-day JWT is cached locally (`~/.tranzor_exporter_auth.json`, **token only — your password is never stored**), a 🔑 header button re-signs-in on demand, and any mid-session `401` auto-prompts and retries. Sign-in is deferred until the window is visible, so the **"Loading… (未响应)" splash no longer freezes** waiting on a hidden dialog.

- ✅ **NEW: Pre-Translation Check tab** — Before you start a manual File translation, import the l10n portal **Purchase delta XLSX** (sheet `Data`: Key = OPUS ID, Value = source) and instantly see what Tranzor has **already** translated, so you never redo or overwrite existing work. Every string is classified **🟢 GREEN — skip** (covered by a merged MR, all needed languages, score OK) · **🟡 AMBER — review** (File/Scan-only, MR not merged, missing langs, or low score) · **🔴 RED — manual** (no Tranzor coverage at all). It joins Purchase OPUS IDs against the **local `opus_index`** (works offline), resolves merge state via GitLab and scores via the platform API. Defaults to **MR-only** coverage, with a one-click toggle to also count File/Scan tasks as skippable. **Export** the manual subset or one-click **copy all 🔴 RED** OPUS IDs.

- 📧 **Full Email review tooling** — Two new userscripts for the File Translation **"Full Email"** viewer, whose nested scroll containers used to make screenshots ghost and the two panes drift apart:
  - **Export PNG / Clean view** — *Export PNG* un-caps both panes and rasterizes the bilingual two-column layout in a single pass (no scroll-and-stitch artifacts); *Clean view* clones the content into a standalone, full-height tab with its own **Print / Save-as-PDF** button.
  - **Synchronized scrolling** — Source and Translation panes now scroll together *proportionally* (tolerant of translation-length expansion), with **Alt+S** to toggle on/off. Targeting is structural, so both also cover the same modal on MR / Scan surfaces if it ships there.

- ✏️ **Post-edit detection — hardened end-to-end** — The ✏️ human-post-edit signal is now faster, more complete, and more honest across all channels:
  - **New "✏️ Post-edited only" filter** on the MR Pipeline list — hide every non-post-edited row and review just the human-edited MRs; more are revealed as the background prefetch confirms them, with a live count.
  - **~10× faster probe** — the per-MR post-edit check now makes **one** GitLab call (was two) over a shared, connection-reused client, so a 50-MR page drops from **~56s → ~5s**.
  - **Branch-scan detection** — the Changes report now scans the MR **source branch** for Language-Lead fix commits, catching UNS email-template and `BATCH_FIX` edits that leave the `fixed_by_lead` audit field NULL (previously missed entirely).
  - **Sticky-badge fix** — Search / Reset / Refresh now drop the cached badge state on **MR Pipeline** *and* **Scan Tasks**, so an edit made later in the Tranzor dashboard lights the ✏️ badge when you come back — a cached "no edit" is no longer permanent.

- 🔢 **en-US Strings column (sortable)** — The MR Pipeline table now shows the **distinct en-US source-string count** per task — the real workload measure, versus the inflated ~18-languages row count — between *Status* and *Avg Score*. Counts fill in asynchronously and are cached; **click any column header to sort**, with the string column defaulting to biggest-first and blank/loading cells always sinking to the bottom.

- 🩹 **Human Revisions windowed by *revision* time** — File Translation edits are now found by scanning **every** Completed task and filtering on each edit-log's own timestamp, so a Language-Lead fix on a months-old task (e.g. a March drop fixed in June) finally surfaces. The Date filter now means **"revision date," not "task-creation date."**

- 🛠️ **Send-to-Tranzor & export fixes** — From MR Pipeline HTML reports, the **Changes** export now opens the correct MR Pipeline task (it was misrouting to File Translation), and **All translations** ticks every selected row — a short OPUS ID no longer substring-matches a longer sibling and steals its tick (the "20 selected → 18 ticked" bug). The **Changes HTML export no longer hangs** on a slow terminology service: highlighting now has a hard deadline and skips oversized whole-file (UNS) sources.
